# Interface Funcional

Interface funcional é uma interface que segue o padrão SAM (Single Abstract Method). Isso significa que
esta interface pode conter APENAS um único método abstrato, mas, pode conter outros métodos que sejam
static ou default.


@Functional Interface não é um requisíto para que o compilador reconheça a interface como sendo
uma interface funcional, mas apenas uma ajuda para capturar a intenção do projeto e obter a ajuda
do compilador na identificação de violações acidentais da integração do projeto. Fonte: docs API

# Lambdas

São funções anônimas usadas para simplificar a implementação de interfaces funcionais. Antes do Java8,
interfaces funcionais precisavam ser implementadas usando classes anônimas, era a principal maneira
de passar comportamentos como parâmetros.

Lambdas são funções sem nome (anônimas), criadas para serem usadas num ponto específico, geralmente
onde uma implementação rápida e temporária de uma interface funcional é necessária.

O principal benefício é que ao invés de criar uma classe ou método completo, você pode criar
uma função lambda diretamente no local onde ela será utilizada.

Outro benefício é o reuso da função, pois são funções específicas que contém uma única responsabilidade

Podem ser passadas como argumentos para métodos que esperam interfaces funcionais
